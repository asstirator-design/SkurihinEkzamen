День 4 - Риггинг: FK, IK и настройка рига

Цель дня - сделать риг удобным для анимации: понять FK/IK, добавить контроллеры и настроить ограничения.

FK и IK

  • FK (Forward Kinematics) - вращение каждой кости по цепочке
  • IK (Inverse Kinematics) - движение через цель-контроллер
  • IK удобен для ног/рук, FK - для позвоночника и верхней части

IK для ног: базовая настройка

  1. В Edit Mode создай кость-контроллер у стопы (`foot_ik`)
  2. В Pose Mode выбери кость голени
  3. Добавь IK Constraint
  4. Target: арматура, Bone: контроллер
  5. Chain Length: 2 (бедро + голень)

Контроллеры

  • создавай отдельные кости для управления
  • не анимируй лишний раз технические кости
  • располагай контроллеры рядом с управляемой частью

Constraints

На этом этапе достаточно уверенно использовать IK Constraint и понимать, что ограничения помогают сделать движение контролируемым.

Проверка рига

  • проверь движение ног через IK
  • проверь вращение рук и корпуса через FK
  • поставь персонажа в несколько поз и проверь деформации

Итог дня

  • есть рабочий риг для анимации
  • есть понимание FK и IK
  • персонаж готов к следующему этапу - анимации

Суть урока

На этом этапе ты делаешь риг удобным для анимации. Важно не просто создать скелет, а научиться управлять им.

  • понимание FK и IK
  • настройка управления конечностями
  • добавление контроллеров
  • подготовка рига к анимации

В конце у тебя должен быть персонаж, которым удобно управлять и ставить в позы.

Что такое риггинг

Риггинг — это настройка системы управления персонажем.

Скелет (арматура) уже есть, но без риггинга он неудобен для анимации:

  • движения получаются неестественными
  • сложно управлять персонажем

Задача риггинга — сделать управление простым и понятным.

FK (Forward Kinematics)

FK — это управление “по цепочке”.

Ты вращаешь каждую кость отдельно:

  • плечо
  • предплечье
  • кисть

Особенности FK

  • подходит для плавных движений
  • хорошо работает для рук и позвоночника
  • требует больше ручной работы

IK (Inverse Kinematics)

IK — это управление через цель. Ты двигаешь одну точку, и вся цепочка подстраивается.

Пример: двигаешь кисть → двигается вся рука

Особенности IK

  • удобно для ног и рук
  • быстро задавать позы
  • идеально для ходьбы и бега

Настройка IK для ног (детально)

Шаг 1 — Создание контроллера

  1. Перейди в Edit Mode арматуры
  2. Добавь новую кость
  3. Размести у стопы
  4. Назови `foot_ik.L`

Шаг 2 — Добавление IK

  1. Перейди в Pose Mode
  2. Выбери кость голени
  3. Открой Bone Constraints
  4. Добавь IK Constraint

Шаг 3 — Настройка

  • Target → арматура
  • Bone → foot_ik
  • Chain Length → 2

Результат: движение контроллера управляет всей ногой.

Контроллеры (подробнее)

Контроллер — это кость для управления ригом.

  • не трогаешь основные кости напрямую
  • управляешь через специальные кости
  • делаешь анимацию проще и чище

Constraints

Constraints помогают ограничивать и направлять движение костей.

  • IK Constraint — основной инструмент на этом этапе
  • позже можно добавлять ограничения вращения и осей

Проверка рига (расширенно)

  • проверь IK ног
  • проверь FK рук и позвоночника
  • поставь персонажа в разные позы
  • убедись, что нет сильных деформаций

Упрощённый подход

Не нужно делать идеальный риг. Главное — чтобы он работал.

Цель: персонаж должен быть удобен для анимации.

Задание

  1. Настрой IK для ног
  2. Добавь контроллер
  3. (по желанию) сделай IK для рук
  4. Проверь FK на руках и позвоночнике
  5. Убедись, что персонаж двигается и позируется

Итог

  • есть рабочий риг
  • настроен IK (минимум для ног)
  • понятие FK и IK закреплено
  • персонаж готов к анимации
